## 4. Model Selection

One stratified 5-fold CV pass over the full **experiment registry** (multiple cells, one logical evaluation). Each experiment uses the feature source it was designed for (`tfidf`, `handcrafted`, or `emb`); fold splits and reporting are shared so results are comparable.

**Registry (per plan):** **starting-point** hyperparameters (not grid search): milder **Ridge** `alpha` than sklearnŌĆÖs default for high-dim text; **Lasso/ElasticNet** `alpha` + `max_iter`/`tol` suited to sparse TF-IDF and convergence; **LinearSVR** `C`/`max_iter`/`dual`; **MLP** after SVD with early stopping; **XGB/LGBM** use **library defaults** for capacity (depth, trees, leaves, learning rate) ŌĆö only `random_state`, `n_jobs`, and **`tree_method="hist"`** for XGB (efficient on wide sparse data without shallow trees). **`verbosity=-1`** for LGBM silences logs. For tree models, fold features are fed as **numpy arrays** so training stays stable without feature-name plumbing noise.
- **Random baseline:** `random_baseline_U01` ŌĆö uniform random score in `[0, 5]` (ignores features). Expect QWK Ōēł 0 and worse RMSE/MAE than real models.
- **Baseline:** `StandardScaler` + `Ridge` on the four handcrafted columns only (`handcrafted_Ridge`).
- **TF-IDF track:** `Pipeline([TfidfVectorizer, model])` for Ridge, Lasso, ElasticNet, LinearSVR, XGBoost, LightGBM, MLP (**RandomForest dropped** ŌĆö too slow on 20K sparse features with little gain). **MLP** uses **`TfidfVectorizer` ŌåÆ `TruncatedSVD` ŌåÆ `MLPRegressor`**.
- **Embedding track:** same model set on 384-d embeddings; **StandardScaler + model** for linear / MLP; **no scaler** for XGB / LGBM.

**CV config:** `StratifiedKFold(n_splits=5, shuffle=True, random_state=42)`.

**Metrics (per fold, then averaged):** **QWK** = quadratic weighted kappa on **rounded/clipped** predictions in 0ŌĆō5 (fixed label set `{0,ŌĆ”,5}` for sklearn). **RMSE** and **MAE** compare **continuous** predictions to integer `y` (regression error). Progress prints show **mean QWK, RMSE, MAE** across folds for each experiment.

**Leakage:** TF-IDF / scalers / SVD fit only on training-fold rows; precomputed features are indexed by fold; `sample_weight` from fold `y_train` only.

**Stratification:** labels 0 and 1 merged into one bin for fold assignment only (rare classes).

## 5. Hyperparameter tuning (TF-IDF + SVD + MLP)

**Scope:** Standalone Optuna tuning for the **TF-IDF ŌåÆ `TruncatedSVD` ŌåÆ `MLPRegressor`** pipeline. It does **not** depend on which model won Section 4; run it whenever `feature_data_by_source["tfidf"]`, `y`, `TFIDF_KWARGS`, and `stratification_bins` are defined (typically after data prep and TF-IDF feature construction). Hyperparameter search lives in the modeling module; text/embedding utilities stay in the shared helpers used earlier.

**Protocol:** **Same TF-IDF settings as Section 4** (`TFIDF_KWARGS`); Optuna does not change vectorizer hyperparameters. For **wall time**, the objective can use a **stratified row subsample** and/or **fewer CV folds** than Section 4 (configured in the Phase A cell)ŌĆömean QWK there ranks configs; for reportable numbers, re-run **full-data CV** with the chosen hyperparameters. Same **balanced sample weights**, `stratification_bins`, and **mean QWK** as the optimization target (rounded/clipped 0ŌĆō5).

**Why two phases:** SVD width and MLP capacity interact; searching both jointly is expensive. **Phase A** fixes SVD at the registry width and uses Optuna **TPE** to explore MLP structure and training hyperparameters. **Phase B** freezes the best MLP from Phase A and uses TPE on the integer **SVD latent dimension** over a sensible rangeŌĆöreducing variance from the initial guess without a huge joint space. If you prefer fully reproducible sweeps with no Bayesian sampler, a **fixed grid** over SVD sizes is a reasonable alternative to Phase B.

Re-run the environment cell after editing the modeling module on disk.

### 5.2 Phase B ŌĆö Bayesian search on `svd_n_components` (MLP fixed to Phase A best)

The MLP is rebuilt from Phase AŌĆÖs best trial (same architecture and training settings, with early stopping left on for stability). **TPE** then searches integer **`svd_n_components`** in `[SVD_N_MIN, SVD_N_MAX]`. If you want a simple, repeatable comparison without Bayesian optimization, evaluate a **hand-picked grid** of SVD sizes using the same CV protocol instead of this search.
